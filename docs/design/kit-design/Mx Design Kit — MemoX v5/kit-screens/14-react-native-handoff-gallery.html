Mx React Native Handoff Gallery

The implementation contract for React Native agents. MemoX v5 is Expo SDK 57 · React Native · TypeScript · NativeWind. Every kit name maps 1:1 to a token module, a theme module, or a shared component; every shared component maps to a React Native primitive. Follow the guard rules and no screen will need a raw style.

Authoritative handoff This page is the single source of truth for React Native work. The base kit's earlier Flutter handoff docs conflicted with the real Expo/RN stack and have been removed; every reference now points here (see DRIFT-002 — RESOLVED in the repo drift log). Token modules

The tokens/*.css variables become typed TypeScript modules. Screens import these — never a literal value. In NativeWind they surface as approved Tailwind theme keys; in StyleSheet as constants.

Theme modules

ModuleResponsibility
MxThemeProviderWraps the app once at the root; supplies the resolved MxTheme via context.
MxThemeResolverMaps MxThemeMode + OS colour scheme → the concrete MxDarkDefault / MxLightDefault theme object. The only place a mode → theme decision happens.
MxThemeModeEnum: system · dark · light. User preference persisted; default system.
MxDarkDefaultThe primary theme object (dark :root token values).
MxLightDefaultThe production-ready light theme object ([data-theme="light"] values). Not an inversion.
useMxTheme()Hook returning the active MxTheme. The one sanctioned way a component reads colours.
// App root — resolve the theme ONCE. <MxThemeProvider mode={MxThemeMode.system}> <MxAppShell>{/* navigation + screens */}</MxAppShell> </MxThemeProvider> // Any component — read colours from the resolved theme. function DeckCount() { const theme = useMxTheme(); return <MxCard style={{ backgroundColor: theme.color.surface.card }} />; }
Resolve mode → theme once in MxThemeResolver; read colours via useMxTheme(). Theme change is visual only — it never touches SRS, sessions, or stored data.
Call useColorScheme() in a screen and branch into raw colours, or duplicate a dark and a light style block inside a component.

Guard rules

These are the review gates for any React Native screen. Each maps to a forbidden pattern.

Raw hex colour in a screen style (color: '#4b3a8c').
theme.color.action.primary via useMxTheme().
Arbitrary spacing / font size / radius (padding: 13, fontSize: 15.5).
MxSpacing.s12, MxTextTokens.bodyMedium, MxRadius.lg.
A one-off StyleSheet that re-implements a kit component (custom card, custom button, custom settings row, custom progress bar).
Use MxCard / MxActionButton / MxSettingsRow / MxStudyProgressBar.
useColorScheme() branching into raw colours, or duplicated dark/light style blocks inside a screen.
One MxThemeProvider / MxThemeResolver; components read useMxTheme().
A bare Pressable styled as a button, or an inline style array with raw values for reusable UI.
MxActionButton variant="…" — every tappable text action.

If the implementation uses NativeWind

Expose only mx-approved token classes; forbid arbitrary values.

Approved token aliases only: p-mx-16, rounded-mx-lg, bg-mx-surface-card, text-mx-body-md — generated from the mx tokens in the Tailwind theme config.
Arbitrary values: p-[13px], text-[#123456], rounded-[11px]. Feature screens must use approved aliases or Mx components — no per-screen Tailwind class soup for reusable UI.
// tailwind.config — theme keys come from mx tokens ONLY. theme: { extend: { colors: { 'mx-surface-card': 'var(--mx-color-surface-card)' }, spacing: { 'mx-16': '16px' }, borderRadius: { 'mx-lg': '16px' }, }} // eslint / tailwind rule: block arbitrary-value class syntax for reusable UI.

If the implementation uses StyleSheet

Every value references an mx token constant: padding: MxSpacing.s16, borderRadius: MxRadius.lg, color: theme.color.text.primary.
Raw numbers (except flex and documented exceptions), raw colour strings, or raw typography values.
const styles = StyleSheet.create({ card: { padding: MxSpacing.s16, borderRadius: MxRadius.lg, borderWidth: MxBorderTokens.width.hairline, // colour comes from useMxTheme() at render, not hardcoded here }, });
Open dependencies Icon set (Expo declares expo-symbols / SF Symbols; kit previews substitute Lucide — confirm the cross-platform strategy) and the chart library for MxStatsChartPanel are unresolved. Do not hardcode a choice; treat both as implementation dependencies to confirm before building Statistics.
