Mx Kit Screens · 06

Mx Learning Component Gallery

Every study surface in MemoX is one of a small set of Mx learning primitives. The same five mode-mechanics — review, match, guess, recall, fill — appear in two flows that share interaction but differ in lifecycle. New Learning (pre-SRS, Box 0) runs the mandatory sequence review → match → guess → recall → fill; completing fill after the prior four activates the card into Box 1. SRS Repeat (Box 1+ due cards only) starts from a mode menu and offers match / guess / recall / fill — no review — and is the only flow that grades and reschedules SRS. Feedback in New Learning is pre-SRS learning, never SRS grading. Sources: docs/design/07-study-modes.md, docs/design/06-srs-8box.md.

Two flows, never confused reviewMode is only the first step of New Learning — it is not in the Repeat Mode Menu. New Learning match/guess/recall/fill is pre-SRS (no box/due change); Repeat match/guess/recall/fill is SRS review for Box 1+ due cards. Correct/wrong feedback and the recall 20s timer are learning signals; SRS scheduling happens only in Repeat, per 06-srs-8box.md.

MxStudyShell — RN handoff contract

Not yet a kit component. MxStudyShell is a React Native handoff contract, not a built specimen in this kit. It describes the session frame (SessionShell in 07-study-modes.md) that hosts every mode. Do not claim a kit component exists for it.

Purpose. The persistent session frame shared by all five modes in both flows: top bar with exit, the mode progress region, the swappable mode body, and the sticky bottom action zone. It owns session I/O and (in Repeat only) the SRS engine call; modes never query the DB.

Anatomy. Top bar (back / exit + mode title) → MxStudyProgressBar → mode body (MxStudyCard + one mode primitive) → sticky bottom actions. Variants: flow="newLearning" | "srsRepeat". States: active, paused/resumable, completed, cancelled, expired, no-due (Repeat with 0 due cards → shows a no-due state, never an empty session).

MxStudyProgressBar

Purpose. Shows progress through the current mode / session — labelled as mode progress, never SRS box mastery. Anatomy: a filled track + a step label (e.g. 3 / 12) and a mode label (e.g. Fill). MxMatchTile

Purpose. A tile in matchMode — pair a prompt (front) with its meaning. The system evaluates only when two tiles are selected: same card → correct, different cards → wrong + retry. Anatomy: a tappable tile; side="prompt" tiles use the Korean font. States: default, selected, correct (green), wrong (red, retry), completed (matched pair locked/hidden). In New Learning it is pre-SRS learning; in SRS Repeat the same mechanic grades a Box 1+ due card.

MxAnswerOption

Purpose. A choice in guessMode — pick the one correct meaning for a prompt among distractors. Anatomy: a full-width option row with label + status glyph. States: default, selected (chosen, not yet judged — violet), correct (green + check), wrong (red + cross), disabled. The critical distinction: selected is not correct, and wrong is not destructive. Same mechanic in both flows — pre-SRS learning in New Learning, SRS grading in Repeat.

MxRecallPanel

Purpose. The recallMode body: show the prompt, hide the meaning, run a 20s reveal countdown, then let the learner self-grade. Anatomy: a Show button carrying the live countdown → on reveal, the meaning + Forgot / Remembered buttons. Phases/states: hidden (countdown running) → revealed (self-grade). Timeout without pressing Show counts as Forgot; Remembered is only valid if Show was pressed before timeout.

MxFillInputPanel

Purpose. The fillMode body — the strongest test and, in New Learning, the SRS activation gate: show the meaning, type the prompt back, Hint + Check. Anatomy: a text input + Hint + Check; on wrong, the expected answer is shown and Retry appears. States: idle, correct (green), wrong (red, shows expected + Retry). A correct check after the prior four modes activates Box 0 → Box 1; a wrong check never activates.

MxPlayMenu — RN handoff contract

Not yet a kit component. MxPlayMenu is a React Native handoff contract, not a built specimen. It maps the deck/subdeck Play Menu entry points to flows.

Purpose. Opened from a deck/subdeck Play action; presents the product-level entry points. Anatomy: a list/sheet of options. Learn → New Learning Flow (always starts reviewMode); Repeat → opens MxRepeatModeMenu (below); Review Words → browse (no session); Game / Player are Future. States: per-option availability from counts — e.g. Repeat shows a due count that may be 0 (progress > 0% does not imply due cards).

MxRepeatModeMenu — RN handoff contract

Not yet a kit component. MxRepeatModeMenu is a React Native handoff contract, not a built specimen. Tapping Repeat opens it; SRS Repeat starts only after the user picks a mode.

Purpose. Let the user choose how to review Box 1+ due cards. Anatomy: four mode choices — Match / Guess / Recall / Fill. Crucially, reviewMode is NOT here — it is only the first step of New Learning. States: a mode is chosen → the SRS Repeat session begins; if due count = 0, a no-due state is shown (no empty session, no SRS mutation).

Business visual rules

These learning-specific rules are binding. They keep interaction states, flows, and SRS meaning from collapsing into each other. Grounded in 06-srs-8box.md and 07-study-modes.md.

Selected is a pending, ungraded pick — state-selected (violet). It is not correct.
Render selected in correct-green, implying the answer was already judged right.
Wrong uses feedback-wrong — a learning outcome the user can retry.
Style wrong with destructive-* — a wrong answer is not data loss.
Keep pre-SRS (Box 0, New Learning) visually distinct — no box/due, feedback is learning only. fill-correct after 4 modes activates Box 1.
Confuse pre-SRS New Learning with SRS Repeat — only Repeat grades and reschedules Box 1+ due cards.
Show only Match / Guess / Recall / Fill in the Repeat Mode Menu.
Put reviewMode in the Repeat Mode Menu — review is New Learning's first step only.
Treat no-due as neutral/positive (srs-noDue green) — the user is caught up.
Show no-due as an error state or auto-create an empty session.
Label the progress bar as mode/session progress.
Present progress as SRS mastery unless a screen explicitly shows box state.
