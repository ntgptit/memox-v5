Mx Kit Screens · 04

Mx Spacing & Layout Gallery

Every MemoX screen breathes on one rhythm. The gaps are not chosen per screen — they are --mx-space-* and --mx-layout-* tokens on a 4px grid, applied the same way at every altitude: screen gutter, section gap, item gap, compact row gap, and the small insets inside a row. This page fixes the measured value for each relationship, shows it on the same simple screen across three device widths so the rhythm reads identically, and states the React Native layout rules that keep it consistent — safe area, keyboard, and flex over absolute positioning.

Source of truth The measured gaps below resolve from tokens/spacing.css (--mx-space-*, --mx-size-*, --mx-layout-*). A screen never invents a gap — if a needed value is missing, add a token, never a raw pixel.

React Native layout notes

The rhythm above is enforced by a few structural rules. These are handoff contracts, not suggestions — a screen that breaks them will drift off the grid.

Safe area lives in the shell The SafeAreaView / safe-area wrapper belongs in MxAppShell / MxScreen — not in every screen. Screens receive already-inset content and never re-apply top/bottom insets themselves.
One scroll owner per screen Do not nest a ScrollView or FlatList inside another scroller. Pick one scroll container; sticky regions (top bar, bottom action) sit outside it, not inside a second scroll view.
Bottom action respects the inset The sticky action band pads with paddingBottom: 16 + safeAreaInset.bottom (the 16 is --mx-space-16; the inset comes from useSafeAreaInsets()). It must never overlap the Android gesture bar or the iOS home indicator.
Keyboard-aware forms are required Fill / input / settings forms wrap in KeyboardAvoidingView (behavior padding on iOS, height on Android) so the focused field and its action lift above the keyboard. Never leave an input hidden behind the keyboard.
Flex, not absolute Compose with flex (flexDirection, gap, flex: 1) consistently. Avoid position: 'absolute' unless a specific recipe requires it (e.g. an overlaid badge). Absolute positioning breaks the shared rhythm and reflow.
// MxScreen: the shell owns safe area + gutter; screens just fill it <SafeAreaView edges={['top','bottom']} style={{ flex: 1, backgroundColor: theme.color.background.app }}> <View style={{ flex: 1, paddingHorizontal: MxLayout.screenPaddingX }}> <FlatList /* the ONE scroll owner */ contentContainerStyle={{ gap: MxSpacing.s12 }} // item gap ListHeaderComponent={<SectionBlock />} // sections use gap: MxSpacing.s24 /> </View> <View style={{ paddingTop: MxSpacing.s16, paddingBottom: MxLayout.bottomActionInset + insets.bottom }}> <MxActionButton label="Start studying" /> </View> </SafeAreaView>

Do & don't

The most common ways a screen falls off the rhythm — each bad example is labelled .mx-dont with a matching visual.

Section rhythm

Give sections room: --mx-space-24 between blocks, items inside at --mx-space-12. The eye groups by the larger gap.
Cram sections at the same gap as items — the screen reads as one undifferentiated pile with no grouping.

Consistent vs random spacing

Every gap is a token step (12 / 12 / 12). One rhythm repeats down the list.
Eyeballed, off-grid gaps (4 / 20 / 8). The list looks broken even though each row is fine.

Nesting

Keep one card level. Group inside a card with spacing and a hairline divider, not a second bordered card.
