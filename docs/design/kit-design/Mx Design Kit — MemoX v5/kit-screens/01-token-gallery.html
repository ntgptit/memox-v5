Mx Kit Screens · 01

Mx Token Gallery

Every visual value in MemoX is a token defined once in tokens/*.css and consumed through var(--mx-*). Screens never hardcode colour, spacing, or type. Each group below shows the token, a live sample (dark and light), its usage, a do/don't, and the React Native style it maps to. Live swatch specimens are the base kit's own token cards, embedded here unchanged.

Source of truth Values shown are the resolved token values from tokens/colors.css · typography.css · spacing.css · radius.css · elevation.css · motion.css. If a value you need is missing, add a token — do not hardcode it.

Colour tokens

Primitive palette (raw hues) → semantic roles (surface, text, action, state) → feedback, SRS, destructive, and chart roles. Screens use semantic tokens, not primitives.

Use the semantic role that names the intent: feedback-correct for a right answer, srs-due (amber) for due cards, destructive-* only for data loss.
Reach for a primitive (--mx-green-500) in a screen, or reuse wrong red for a destructive action — they are different roles even when the hue is close.
React Native Colours resolve from MxTheme via useMxTheme() — never a literal hex and never a useColorScheme() branch inside a screen. Example: { backgroundColor: theme.color.surface.card }.
