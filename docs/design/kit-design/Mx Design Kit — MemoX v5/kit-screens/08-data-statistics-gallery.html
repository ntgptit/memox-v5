Mx Kit Screens · 08

Mx Data & Statistics Gallery

The data surface of MemoX is deliberately honest: MxMetricCard shows a single finalized number, MxStatsChartPanel renders a per-day trend of real history, and MxSessionResultSummary reports only the phiên that just finished. Statistics is read-only — opening it never mutates learning data, never schedules SRS. Where a formula or aggregation isn't defined yet, these components must show a no-data or read-model-needed state — never fabricate a line or a percentage.

Concept spec, not fake data Per docs/design/screens/statistics.md, exact aggregation formulas (Đang học / Đã học / Đã lặp lại, time totals, quality ratios) are owned by the read-model tasks (F1.STATS.4–6). Until a formula lands, the panel shows a read-model-needed state — it does not invent numbers.

MxSessionResultSummary

Purpose — the end-of-session summary shown after a New Learning or SRS Repeat flow finalizes. It reports only the phiên that just finished and is strictly read-only: it never activates cards into Box 1 and never schedules SRS. It is not Statistics — it may link to Statistics but does not replace it.

Session Result ≠ Statistics Per docs/design/screens/session-result.md, Result covers a single finalized session; long-term trends live on the separate Statistics screen (MxStatsChartPanel). Result must not fabricate a composite score or progress % — those formulas are open questions.

React Native Handoff name MxSessionResultSummary; props { type?, headline, subhead?, stats? } where each stat carries a tone of correct | wrong | active | due. Numbers use the --mx-text-stat-number mono tabular token; the component reads a finalized session read-model and performs no writes.

Rules — honest data, read-only statistics

Charts and tiles render only real, finalized data. With none, show the friendly empty state or the read-model-needed state.
Present fake / placeholder data as real — no invented lines, interpolated gaps, or guessed percentages.
When a formula/aggregation is undefined, show the read-model-needed state and cite the owning task (F1.STATS.4–6).
Let opening Statistics mutate learning data — it is read-only: no box change, no due-date change, no SRS scheduling.
Keep Session Result (one finished session) distinct from Statistics (long-term, many sessions); Result may link to Statistics.
Treat Session Result as Statistics, or mix New Learning activation with SRS Repeat metrics in the same reading.
Open dependency — chart library The chart type/library is not chosen yet. This page shows the visual contract (tokens, grid, axis, series colours, empty + read-model-needed states); the implementation dependency is OPEN (candidates: victory-native, react-native-svg). Do not hard-code a library into screens — depend on the MxStatsChartPanel contract.
