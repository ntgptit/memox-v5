Mx Kit Screens · 11

Mx Layout Template Gallery

The nine named layout templates every MemoX screen is built from. A template is a region skeleton — a composition of Mx shells + slots + token spacing — not a new component and not a final product screen. Each template below renders its regions in dark and light on a phone frame, with anatomy and spacing labels, plus a spec for slots, scroll, safe-area, keyboard, and allowed action placement. Screens fill the slots with Mx components and never re-implement the shell.

These are templates, not screens The skeletons show region contracts: top bar, scroll body, sticky bottom action, section rhythm. Real screens (golden references, embedded at the end) fill each region with concrete Mx components. Reference device is the Samsung Galaxy S23 Ultra (19.3:9, CSS 384×824, Android-first).

React Native primitive mapping

Each region concept above maps to a fixed React Native primitive and a fixed Mx shell/component that wraps it. Templates never re-declare these — they compose the shells and fill the slots.

React Native handoff Templates are the region contract: a screen picks a template, fills each slot with Mx components, and never re-implements the shell. Bottom actions respect SafeAreaView insets (--mx-layout-bottom-action-inset); any form (Study fill, Settings editors) wraps its inputs in KeyboardAvoidingView so the field and its primary action stay above the keyboard. One primary per region, placed near the bottom; destructive uses destructive tokens, never a normal primary.
