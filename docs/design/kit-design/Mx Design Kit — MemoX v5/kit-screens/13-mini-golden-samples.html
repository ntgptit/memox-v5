Mx Kit Screens · 13

Mx Mini Golden Samples

Five miniature composition exemplars — a list, a study session, a settings hub, a modal, and a statistics view — each built only from Mx components and mx tokens. They show how the pieces sit together: row rhythm, action hierarchy, state separation. These are reference compositions, not final product screens — deck/flashcard/language lists, the five study modes, SRS Repeat, and the real Settings/Statistics screens are built by mimicking these shapes, not by copying this content. React Native is the only target; there are no Flutter concepts here.

Exemplar, not spec Content below (KO→EN cards, the deck "TOPIK I Core 800", "Due 83", "Nothing due") is illustrative. The structure — regions, spacing rhythm, one-primary rule, and state coverage — is what future screens copy. Each sample pairs a dark loaded phone with a light (data-theme="light") loaded phone, plus one small frame in a non-happy state.

Action hierarchy

New Learning active Repeat disabled · no due Reset destructive · separated
Disable (not hide) SRS Repeat when nothing is due and say why; separate the destructive "Reset" below a divider so it can't be tapped by momentum.
Mix the destructive row into the normal group, or leave a disabled row looking tappable — dim it and mark the reason ("Nothing due" / "Soon").
React Native MxModalSheet hosts a column of MxListRow with a disabled prop for no-due/unavailable and intent="destructive" for Reset. Repeat's disabled state is driven by the due count (local-day rule), not hidden. The grabber, sheet radius, and shadow all come from MxModalSheet tokens — an RN agent wires rows and intents, no raw style.

Use tabular mono for every count and percent so columns align; colour chart series from chart-* roles and show the honest empty state when the read model is empty.
