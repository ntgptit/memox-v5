Mx Kit Screens · 03

Mx Typography Gallery

Three families — Plus Jakarta Sans (UI / display / body), Noto Sans KR (Korean prompts), JetBrains Mono (numbers) — resolved through --mx-text-* and --mx-font-* tokens. A screen never sets a raw fontSize; it picks a role. Below: the base kit's type specimen cards (embedded unchanged), then live KO / VI specimens per text role — each with its token, an example, the max-line rule, the wrap/truncate rule, and the React Native <Text> it maps to, shown dark and light.

Source of truth Sizes are the resolved values from tokens/typography.css. Korean prompt is 34/44, meaning wraps at 18/28, body floor is 13px. If a role you need is missing, add a token — never hardcode a px size in a screen.

Deck title — truncates to one line

A deck / list-row title is a single line that truncates with an ellipsis — it never wraps to a second line or pushes the trailing meta out of the row.

Typography rules

How each role behaves under long KO / VI content and OS text scaling.

Keep the Korean prompt highly readable — full study-korean size, centred, high contrast — and let the Vietnamese explanation wrap comfortably at study-explanation.
Shrink long text below the accessibility floor (body 13px, study 34px) to make it fit — or clip Korean / Vietnamese with an ellipsis.
Wrap long content and put it in a ScrollView; truncate only single-line labels (deck titles, list rows) with a tail ellipsis.
Let RN <Text> overflow or clip its container — for long KO / VI, prefer wrapping + a scrollable area over tiny type.
Accessibility floor Body text never renders below 13px and the study prompt never below 34px, even when content is long. Length grows the height (and the scroll area), never shrinks the type.
