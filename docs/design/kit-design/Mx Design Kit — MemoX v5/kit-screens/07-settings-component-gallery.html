Mx Kit Screens · 07

Mx Settings Component Gallery

The Settings family is where MemoX exposes preferences without ever mutating learning data. Every line on a settings screen is an MxSettingsRow variant — never a hand-rolled row — grouped by MxSettingsSection, with toggles rendered as MxSwitch. This page documents each built component (anatomy, states, tokens, accessibility, and the React Native handoff), flags the four rows that are still RN handoff contracts, not kit components, and shows the high-risk / unavailable states in token-styled dark+light demos. The live specimen below is the base kit's own settings card, embedded unchanged.

Opening ≠ mutating Reaching a settings screen or a child settings screen never changes decks, cards, sessions, box, or due_at. Preference changes take effect going forward only. This is a hard product contract, not a visual nicety — see docs/design/09-settings.md and docs/design/screens/spaced-repetition-settings.md.

Live component gallery

Settings section + rows, native-language rows, a reminder row with weekday chips, and theme preset tiles — rendered from the real Mx* components in both dark (primary) and light themes.

MxSettingsSection

A titled group of settings rows. It owns the section label, an optional descriptive line, and the shared card surface / dividers around its children. It carries no interaction of its own — it is a layout + grouping primitive.

MxSettingsRow

The settings-line primitive: leading glyph, title + optional subtitle, and a trailing value (text) or control (e.g. an MxSwitch). It reflects and navigates; opening never mutates. Every settings line is a variant of this row — the app never hand-rolls a settings row.

MxSwitch

The on/off toggle used inside a settings row's control slot. It is the only toggle in the kit — a settings boolean is never a checkbox or custom tap area.

MxReminderRow

One reminder line: a tappable time, a strip of MxWeekdayChips, an enable MxSwitch, and a delete affordance. It never starts a session or mutates learning data — it edits a notification schedule only. invalid (no weekday selected) shows an inline warning outline.

RN handoff contracts — not yet kit components

The following rows are documented contracts for the React Native build. They do not exist as kit components yet — each is expected to be a variant of MxSettingsRow (never a hand-rolled row) once built. The demos below are token-styled illustrations of the intended shape, not shipped components.

MxAppIconTile contract — not built

An app-icon variant tile — conceptually a specialised MxThemePresetTile whose preview is an icon rather than a theme swatch. Icon variants a platform can't apply render unavailable. No such component exists in the manifest yet.

Rules

Treat opening a settings screen (or child SRS-settings screen) as read-only navigation — it reflects state and never mutates decks, cards, sessions, box, or due_at.
Perform any write on screen open or row tap. Preference changes apply going forward and, for high-risk data operations, only after confirmation.
Make high-risk data operations visually distinct with --mx-color-destructive-* — destructive text, surface, and a 1.5px destructive border, plus a glyph and "permanent / overwrites" copy.
Style "Delete all decks" or "Restore from backup" like an ordinary row, or reuse answer-wrong red for it — destructive is its own role.
Make future / unavailable rows look intentional — a "Soon" pill on --mx-color-state-unavailableSurface, so "coming soon" reads as a decision.
Let an unavailable row look broken, error-coloured, or half-rendered — an empty/greyed row with no label is a bug, not a "Soon" affordance.
Render the SRS "Ô: 7" box setting as an OPEN QUESTION — amber, blocked, awaiting product decision — consistent with 8-box being the source of truth.
Ship "Ô: 7" as a final 7-box rule, or migrate the 8-box model to 7 boxes to match the mock. That contradicts 06-srs-8box.md.
React Native Every settings line is an MxSettingsRow variant — never a hand-rolled row. Toggles are MxSwitch; grouping is MxSettingsSection. Destructive rows (tone="destructive") route their action through MxConfirmDialog — the row opens the confirm, the dialog performs the mutation. Unavailable rows (unavailable) show "Soon" and set accessibilityState={{ disabled: true }}. The SRS "Ô: 7" box setting stays a blocked open-question until a product decision lands.
