Mx Kit Screens · 09

Mx Feedback & Dialog Gallery

How MemoX speaks back to the learner — a quiet line of context (MxCallout), a transient confirmation (MxSnackbar), and a deliberate stop for risky actions (MxConfirmDialog). The through-line is calm: feedback names the intent with a semantic role, reassures rather than alarms, and reserves visual weight for the one place it matters — data that could be lost. Each block below shows purpose, anatomy, states, the tokens in play, accessibility, and the React Native component it hands off to, dark and light.

Reassurance over alarm Colour signals intent, never volume. A card being due is amber, not red; nothing due is a calm positive; a failed finalize keeps the learner's data and offers a retry. Only real data loss earns destructive red. If copy needs an exclamation mark to land, the copy is wrong — rewrite it calm.

MxSnackbar

Purpose. A transient bottom toast that confirms a completed action or reports a quick outcome, with at most one inline action (typically Undo). It auto-dismisses and never blocks the task — for anything the learner must decide, use MxConfirmDialog instead.

Keep it to one line and one action; auto-dismiss; use tone="success" with an Undo action for reversible deletes.
Stack multiple snackbars, put two actions in one, or use a snackbar to ask a destructive question — that is a dialog, not a toast.
Accessibility Announce via aria-live="polite" (assertive only for errors). Icon is decorative — tone must also read from text. Dismiss timing respects reduced motion; the message never depends on colour alone.

MxCallout

Purpose. An inline context banner that stays in the layout to explain a condition — an informational note, a caution before a settings change, an inline destructive warning, a calm "nothing due", or an unavailable/coming-soon area. It informs; it does not interrupt.

tone → role → token
toneMeaningSurface / border token
infoNeutral context or tip--mx-color-feedback-infoSurface · --mx-color-feedback-info
warningProceed-with-care caution--mx-color-feedback-warningSurface · --mx-color-feedback-warning
destructiveInline data-loss warning — visually stronger--mx-color-destructive-surface · --mx-color-destructive-border
noDueCalm / positive — nothing due, reassuring--mx-color-srs-noDue
unavailableDimmed, feature not available yet--mx-color-state-unavailableSurface
Use tone="noDue" with reassuring copy ("Nothing due — you've reviewed everything today"); use warning before a reversible settings change.
Style noDue like an error, or use destructive for a mere caution — destructive tone is reserved for actual data loss.
Accessibility Title + body carry the meaning; the tone glyph is decorative. Contrast holds in both themes. A callout is static context, not a live announcement, so it needs no aria-live — but destructive callouts should sit next to the control they warn about.
React Native handoff <MxCallout tone="noDue" title="Nothing due">You've reviewed everything today.</MxCallout>. Props: tone (info | warning | destructive | noDue | unavailable), title, children, icon. Renders in-flow inside MxScreen — no overlay, no scrim.

MxConfirmDialog

Purpose. A modal that stops the flow and asks the learner to confirm before an action that is hard or impossible to undo. The variant scales the risk signal so a routine confirm, a destructive delete, and a high-risk overwrite never look the same.

variant · anatomy · token
variantSignalConfirm-button / border token
normalRoutine confirm — primary action styling--mx-color-action-primary
destructiveData loss (delete deck/cards) — destructive red, stronger--mx-color-destructive-text · --mx-border-destructive
highRiskOverwrite / restore — strongest; auto-shows warning line--mx-color-destructive-border · --mx-color-feedback-warning

Anatomy: scrim (--mx-color-overlay-scrim) → modal card (--mx-color-surface-modal, --mx-radius-xl, --mx-elevation-modal) → title → message → optional warning line → cancel + confirm. Cancel is always the safe default; the confirm label is a verb ("Delete", "Restore"), never "OK".

Use variant="destructive" for deletes and variant="highRisk" for restore / cloud overwrite; label confirm with the verb; keep cancel prominent.
Style a destructive or high-risk confirm like a normal primary, or use a raw platform Alert for data-loss actions.
Accessibility Focus traps inside the dialog; initial focus lands on the safe (cancel) action. Role dialog, labelled by the title, described by the message + warning. Scrim tap and hardware back both cancel — they never confirm.
React Native handoff <MxConfirmDialog variant="destructive" title="Delete this deck?" message="All cards in the deck will be deleted." confirmLabel="Delete" onConfirm={del} onCancel={close} />. Props: open, variant (normal | destructive | highRisk), title, message, warning, confirmLabel, cancelLabel, onConfirm, onCancel. Rendered in an RN Modal over a scrim, centred within safe-area insets.

MxFailureMessage handoff contract

Purpose. A reassuring, recoverable failure surface for boundary operations — most importantly finalize-failed and restore-failed. Per the Session Result contract, a failed finalize must never be shown as a completed result and must never silently mark the session complete: the learner's data is preserved and a retry path is offered. This is a handoff contract — the RN component is specified here; its final implementation lands with the feature.

state · copy intent · token
StateCopy intent (calm, no alarm)Token
finalize-failed"Your learning data is safe. We couldn't save this session yet — retry."--mx-color-feedback-warningSurface · --mx-color-feedback-warning
restore-failed"Restore didn't complete. Your current data is unchanged — retry."--mx-color-destructive-surface · --mx-color-destructive-border
actionPrimary Retry + secondary dismiss / go back--mx-color-action-primary
Preserve data on failure and lead with the reassurance ("Your learning data is safe. Retry."); keep the failed session out of the completed-result view.
Discard progress, silently mark the session completed, or present a bare stack trace / "Error 500" with no recovery path.
Accessibility Announced aria-live="assertive" when it appears; the Retry control is the focused, primary target. Warning tone (finalize) and destructive tone (restore) read from text, not colour alone.
React Native handoff MxFailureMessage — props (proposed): kind (finalizeFailed | restoreFailed | generic), title, message, onRetry, onDismiss. Composes an inline surface or an MxModalSheet; data safety is guaranteed by the feature layer, not the component.

MxNoDueState handoff contract

Purpose. The calm "nothing due" surface for when a deck or the day's queue has no cards to review right now. It is neutral / positive, conceptually a variant of MxEmptyState (tone="positive") — distinct from an error state. Nothing is wrong; the learner is simply caught up. This is a handoff contract paired with the noDue callout tone.

Accessibility Static content (not a live announcement); positive tone reads from the copy, not colour. Contrast holds in both themes; the optional CTA meets the 44px touch target.
React Native handoff MxNoDueState — realised as <MxEmptyState tone="positive" title="Nothing due" message="You've reviewed everything today." action={…} />. Props inherit MxEmptyStateProps: icon, title, message, action, tone (neutral | positive | unavailable). Never routed to MxErrorState.

Rules

Present no-due as neutral / positive — "Nothing due", reassuring copy, never an error colour or tone.
Dress no-due in warning amber or destructive red, or imply the learner did something wrong by being caught up.
Make destructive and restore warnings visually stronger than a normal confirm — destructive surface/border, a warning line for high-risk.
Style a delete or restore confirm like a routine primary button, so risk and routine look identical.
On finalize-failed, preserve the learner's data and show a retry path — "Your learning data is safe. Retry."
Discard progress, silently mark the session completed, or show a failed finalize as a completed result.
Reserve destructive red for real data loss; use amber for caution and feedback-info for neutral notices.
Reuse answer wrong red as destructive intent, or lean on colour alone to carry the message.
React Native Confirm and destructive flows use MxConfirmDialog over a scrim (--mx-color-overlay-scrim) — never a raw Alert for destructive actions. Transient feedback uses MxSnackbar (auto-dismiss, announced via aria-live). MxModalSheet / dialog surfaces respect SafeAreaView insets so nothing hides under the home indicator or notch; the bottom sheet honours the bottom inset and --mx-size-sheet-max-height.
