Mx Kit Screens · 05

Mx Core Component Gallery

The core layer is the grammar every MemoX screen is written in: a screen shell, a top bar, one card surface, one list row, one text-action button, one icon-only button, a segmented switcher, a modal sheet, a confirm dialog, and the calm empty / reassuring error states. Every value is a var(--mx-*) token; nothing here hardcodes colour, spacing, or type. The live specimens below are the base kit's own component cards, embedded unchanged in dark and light — the spec blocks add anatomy, states, token usage, accessibility, and the React Native handoff name and props for each.

Built vs. contract Ten of these are built kit components (in the manifest components[]): MxTopBar, MxCard, MxListRow, MxActionButton, MxIconButton, MxSegmentedControl, MxModalSheet, MxConfirmDialog, MxEmptyState, MxErrorState. The four shells — MxAppShell, MxScreen, MxSafeAreaShell, MxContentShell — are React Native handoff contracts named here, not yet kit components. Do not assume a shell exists as base-kit UI.

Live specimens

The base kit's core-component card renders buttons, cards, rows, the top bar, badges, and the empty/error states in both themes. It is the source of truth for pixels — the spec blocks below do not redraw it.

Screen shells RN contract

Four nesting contracts that frame every screen. They are named by the React Native handoff and are not yet built kit components — they describe how the built components compose inside a safe-area-aware, scrollable, single-column screen. Screens never re-invent padding or safe-area insets; they consume these shells.

MxAppShell — RN handoff contract, not yet a kit component

Purpose The outermost app frame: theme provider, root background, and navigation host that wraps every route. Anatomy theme context (useMxTheme) → root <View> at --mx-color-background-app → navigator. Variants / states none beyond the active theme (MxDarkDefault / MxLightDefault); there is no per-screen theme branch. Tokens --mx-color-background-app, elevation + motion tokens for transitions. Accessibility owns nothing tappable; sets the theme so downstream contrast tokens resolve. RN handoff MxAppShell over a View + navigation container; expected state: current theme, resolved once at the root, never inside a screen.

MxSafeAreaShell — RN handoff contract, not yet a kit component

Purpose Applies device safe-area insets (notch, home indicator, status bar) so content and sticky bars never sit under system chrome. Anatomy a SafeAreaView that pads top/bottom by the inset. Variants edges selection (top, bottom, both). States orientation- and device-driven insets; no interactive state. Tokens combines the platform inset with --mx-space-* gutters — inset is additive, never a substitute for a spacing token. Accessibility keeps the first focusable element and sticky actions clear of system bars. RN handoff MxSafeAreaShell over SafeAreaView (react-native-safe-area-context); expected props: edges, children.

MxScreen — RN handoff contract, not yet a kit component

Purpose The per-route screen frame: screen background + the horizontal gutter every screen shares, hosting an MxTopBar and a scrollable content region. Anatomy background <View> → optional MxTopBar → MxContentShell. Variants variant mirrors the top-bar intent (default / study / settings). States loading (renders skeletons), empty (delegates to MxEmptyState), error (delegates to MxErrorState) — the screen designs each applicable state, never just the happy path. Tokens --mx-color-background-screen, --mx-layout-screen-padding-x (16), --mx-space-40 top pad. Accessibility one screen title (from the top bar) as the a11y heading. RN handoff MxScreen over View; expected props: variant, title, children; expected state: status: 'ready' | 'loading' | 'empty' | 'error'.

MxContentShell — RN handoff contract, not yet a kit component

Purpose The scrollable content column with a capped reading width and a consistent bottom inset so the last item clears any sticky CTA. Anatomy a ScrollView whose content is centred and clamped to --mx-layout-content-max-width (520). Variants scroll vs. non-scroll (a study screen may lock its card), and an optional sticky bottom-action inset. States at-top / scrolled (drives top-bar shadow), refreshing. Tokens --mx-layout-content-max-width, --mx-layout-section-gap (24), --mx-space-64 end pad, --mx-layout-bottom-action-inset. Accessibility the scroll region is the main landmark; sticky actions announce after content. RN handoff MxContentShell over ScrollView; expected props: scroll, contentInsetBottom, children.

Contract, not code These four names appear in handoff diagrams and the RN summary table below. Until they are built and added to the manifest components[], treat them as the agreed composition contract, not importable base-kit UI.

MxTopBar

Purpose The screen header primitive — title, optional subtitle, a leading slot (usually a back/close MxIconButton), and a trailing actions slot. Anatomy leading → title + subtitle block → actions. Variants variant: default / study / settings. States default; sticky gains an elevation/hairline once the content below scrolls; there is no pressed/disabled state on the bar itself (its slots own that). Tokens title at --mx-text-title-large, background --mx-color-background-screen, hairline --mx-color-border-subtle, sticky layer --mx-layer-sticky. Accessibility the title is the screen's a11y heading; every icon slot must be an MxIconButton with a label. RN handoff MxTopBar over View + Text; real props: title, subtitle, leading, actions, variant, sticky.

MxCard

Purpose The one surface-container primitive; feedback outlines are driven by state while the base surface stays constant, so selection is never confused with correctness. Anatomy padded surface + optional border + optional elevation, wrapping arbitrary children. Variants variant: default / elevated / interactive. States state: none / selected (2px selected border) / correct / wrong (2px feedback border, base surface kept) / disabled; pressed is only meaningful for interactive via onClick. Tokens surface --mx-color-surface-card, radius --mx-radius-lg (16), --mx-border-selected / --mx-border-correct / --mx-border-wrong, --mx-elevation-low, padding via the padding token number. Accessibility feedback state is paired with an icon/label, never colour alone; an interactive card exposes a role/label. RN handoff MxCard over View (or Pressable when interactive); real props: variant, state, padding (e.g. "12" | "16" | "20"), onClick.

MxListRow

Purpose The list-row primitive for deck/content lists — a leading slot, a title/subtitle block, a trailing slot, and an optional chevron. Anatomy leading → title + subtitle → trailing → chevron. Variants variant: default / error. States default; pressed (via onClick); disabled; error variant tints for a failed row. Tokens single-line height --mx-size-list-row-height (60), two-line --mx-size-list-row-height-two-line (68), title --mx-text-body-large, subtitle --mx-text-body-small at --mx-color-text-tertiary, divider --mx-color-divider-default. Accessibility whole row is one 44px+ target; chevron is decorative and the row carries the label. RN handoff MxListRow over Pressable + View; real props: title, subtitle, leading, trailing, variant, disabled, onClick, showChevron.

MxActionButton

Purpose The one button primitive for MemoX — every text action is a variant of this; no screen hand-rolls a Pressable button. Anatomy optional leadingIcon → label (children) → optional trailingIcon. Variants variant: primary / secondary / ghost / destructive; size: small / medium / bottom (large sticky CTA). States default, pressed (--mx-color-action-primaryPressed), disabled (--mx-color-action-primaryDisabled), and loading (spinner, taps blocked). Tokens primary fill --mx-color-action-primary, label --mx-text-button / --mx-color-text-onAction, pill radius --mx-radius-pill, heights --mx-size-button-height (48) / --mx-size-button-height-bottom (54); destructive uses --mx-color-destructive-*. Accessibility ≥44px target; label is the a11y name; loading/disabled announce a busy/disabled state. RN handoff MxActionButton over Pressable; real props: variant, size, fullWidth, disabled, loading, leadingIcon, trailingIcon, onClick.

Keep one primary per region; make the sticky CTA size="bottom"; route data-loss actions to variant="destructive".
Style a destructive action as a normal primary, or ship a bare Pressable as a text button instead of MxActionButton.

MxIconButton

Purpose A square, icon-only tap target (≥44px) for top-bar and inline actions. Anatomy a single centred icon inside a 44px hit area. Variants variant: ghost / surface / destructive; size: small / medium. States default, pressed, disabled. Tokens icon --mx-size-icon-md (24), target --mx-size-touch-target (44), surface variant --mx-color-surface-card, destructive --mx-color-destructive-text. Accessibility label is required and maps to accessibilityLabel (rendered as aria-label + title) — an icon-only control is invisible to assistive tech without it. RN handoff MxIconButton over Pressable; real props: icon, label (required), variant, size, disabled, onClick.

MxSegmentedControl

Purpose A pill-track tab switcher (Statistics tabs, filter toggles) — a controlled, single-select segment group. Anatomy a pill track holding one segment per option, with a moving selected pill. Variants single style; each option is a string or { value, label, disabled }. States selected segment (--mx-color-state-selectedSurface / --mx-color-state-selected), unselected, per-option disabled, pressed. Tokens track radius --mx-radius-pill, height --mx-size-chip-height (34), label --mx-text-label-medium. Accessibility exposes a tablist/radiogroup semantic; the selected value is announced, not colour-only. RN handoff MxSegmentedControl over Pressable segments in a View; real props: options, value, onChange — controlled, so the parent owns the selected value in state.

MxModalSheet

Purpose The bottom-sheet primitive (Play Menu, Repeat Mode Menu, pickers) — a dismissable overlay anchored to the bottom edge. Anatomy scrim → sheet surface with optional title → children → optional sticky actions row. Variants single sheet form; content-driven height. States open / closed, scrim-press-to-dismiss (onClose), scrolling when content exceeds the cap. Tokens surface --mx-color-surface-modal, scrim --mx-color-overlay-scrim, radius --mx-radius-xl (22), max height --mx-size-sheet-max-height (88vh), shadow --mx-elevation-sheet, layer --mx-layer-modal. Accessibility traps focus while open, restores on close; scrim tap and a cancel action both dismiss. RN handoff MxModalSheet over Modal + View; real props: open, title, onClose, actions, children.

MxConfirmDialog

Purpose A centred confirmation dialog where high-risk never looks like a normal confirm. See the live specimen above for all three variants in both themes. Anatomy scrim → centred card → title → message → optional warning line → cancel + confirm action row. Variants variant: normal / destructive / highRisk (auto-shows the warning line and a destructive-styled confirm). States open / closed; confirm (onConfirm) and cancel (onCancel). Tokens surface --mx-color-surface-modal, scrim --mx-color-overlay-scrim, destructive confirm --mx-color-destructive-*, warning --mx-color-feedback-warning, width --mx-size-modal-width (400), shadow --mx-elevation-modal, layer --mx-layer-dialog. Accessibility alertdialog role; cancel is the default focus for destructive/high-risk so a stray tap never destroys data; copy reassures what is and isn't lost. RN handoff MxConfirmDialog over Modal; real props: open, variant, title, message, warning, confirmLabel, cancelLabel, onConfirm, onCancel.

MxEmptyState

Purpose A calm "nothing here" state — never styled like an error. Anatomy optional icon → title → optional message → optional action. Variants tone: neutral / positive (e.g. "all caught up") / unavailable. States static; the optional action (an MxActionButton) carries its own states. Tokens muted glyph/text --mx-color-text-tertiary, positive tone --mx-color-srs-noDue, unavailable --mx-color-state-unavailableSurface, breathing room --mx-space-48, title --mx-text-title-medium. Accessibility reads as informational, not an alert; the action is a labelled button. RN handoff MxEmptyState over View + Text; real props: icon, title, message, action, tone.

MxErrorState

Purpose A failure state with retry; copy must reassure that data is preserved. Anatomy optional icon → optional title → optional message → action (retry). Variants single form (distinct from the calm MxEmptyState). States static error; the retry action owns its default/pressed/loading states. Tokens warning accent --mx-color-feedback-warning (not destructive red — a load error isn't data loss), surface --mx-color-surface-card, title --mx-text-title-medium, breathing room --mx-space-48. Accessibility announced as an alert; retry is a labelled MxActionButton; message states data is safe. RN handoff MxErrorState over View + Text; real props: icon, title, message, action.

Use amber feedback-warning for a transient failure and say "your cards are safe — tap retry".
Paint a load error in destructive red, or reuse MxErrorState for an empty list — that's MxEmptyState.

React Native contract Every tappable text action is an MxActionButton — never a hand-rolled Pressable with a Text child. Every surface is an MxCard (or a shell that composes it), never a raw View with ad-hoc padding and borders. Every icon-only action is an MxIconButton and must pass label, which maps to accessibilityLabel. Colours, spacing, radii, and shadows resolve from MxTheme via useMxTheme() — no literal hex, px, or useColorScheme() branch inside a screen.
