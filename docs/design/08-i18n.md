# 08 — Đa ngôn ngữ (i18n giao diện)

Phạm vi: **đa ngôn ngữ hóa giao diện** (nút, nhãn, thông báo). **Không** đa ngôn ngữ hóa *nội dung
thẻ* (thẻ là unicode tự do do người dùng nhập). Xem non-goal N4 ở [01-vision](../product/01-vision.md).

## Thư viện

- **`expo-localization`** (SDK 57) để đọc ngôn ngữ/vùng thiết bị.
  Tham chiếu: https://docs.expo.dev/versions/v57.0.0/sdk/localization/
  - `getLocales()` / `useLocales()` → language tag, region, `textDirection`, hệ đo…
- **`i18n-js`** làm engine dịch (nhẹ, phổ biến, đủ dùng). *(Có thể thay bằng `react-i18next` nếu về
  sau cần tính năng nâng cao — quyết định mặc định là `i18n-js` cho gọn.)*

Cài (khi tới lúc code): `npx expo install expo-localization` + `i18n-js`.

## Ngôn ngữ hỗ trợ

- Bắt buộc: **`vi`**, **`en`** (FR-I2).
- **Mặc định**: theo ngôn ngữ thiết bị (`getLocales()[0].languageCode`); **fallback `en`** nếu thiết
  bị dùng ngôn ngữ chưa hỗ trợ.
- Người dùng đổi thủ công trong Settings; lựa chọn **lưu bền vững** trong `app_meta` (khóa `settings.language`).

## Vị trí & cấu trúc

Đặt ở **`src/shared/i18n`** (dùng chung mọi feature; hợp ranh giới vì `shared` không import feature):

```
src/shared/i18n/
├── index.ts              # khởi tạo i18n, hàm t(), đổi locale
├── use-t.ts              # hook useT() cho component
├── locales/
│   ├── en.json
│   └── vi.json
└── __tests__/
    └── keys.test.ts      # kiểm không sót/không thừa khóa giữa các locale
```

## Quy ước khóa

- Phân cấp bằng dấu chấm, gom theo **miền/feature**: `<feature>.<màn/khối>.<phần tử>`.
- Không nhồi câu dài vào key; key mô tả **vị trí/ý nghĩa**, giá trị là câu dịch.

```json
// vi.json (trích)
{
  "common": { "save": "Lưu", "cancel": "Hủy", "delete": "Xóa", "confirm": "Xác nhận" },
  "deck": {
    "create": { "title": "Tạo bộ thẻ", "namePlaceholder": "Tên bộ thẻ" },
    "delete": { "confirm": "Xóa bộ thẻ này và toàn bộ thẻ/bộ con bên trong?" },
    "dueToday": "{count} thẻ đến hạn"
  },
  "study": {
    "scope": { "includeSubdecks": "Gồm bộ thẻ con" },
    "typing": { "placeholder": "Gõ nghĩa…", "correct": "Chính xác!", "wrong": "Chưa đúng" },
    "summary": { "title": "Kết thúc phiên", "correct": "Đúng", "wrong": "Sai" }
  },
  "settings": {
    "title": "Cài đặt",
    "language": "Ngôn ngữ",
    "intervals": "Khoảng ôn (ngày)",
    "lapseRule": { "reset": "Về hộp 1", "stepDown": "Lùi 1 hộp" },
    "newPerDay": "Thẻ mới mỗi ngày"
  }
}
```

`en.json` giữ **cùng bộ khóa**, chỉ khác giá trị.

## Số nhiều & tham số

- **Nội suy tham số**: `t('deck.dueToday', { count })`.
- **Số nhiều**: dùng cơ chế plural của `i18n-js` (khóa `.one` / `.other`). Tiếng Việt không chia số
  nhiều theo dạng từ nên phần lớn dùng một dạng; tiếng Anh cần `one/other`.
- **Định dạng số/ngày** (FR-I4): dùng `Intl.NumberFormat` / `Intl.DateTimeFormat` với locale hiện tại
  (có sẵn trong Hermes/RN hiện đại). Tránh tự ghép chuỗi ngày thủ công.

## API dùng trong app

```ts
// src/shared/i18n/index.ts (hình dạng dự kiến)
export function initI18n(): void;                 // đọc locale thiết bị + locale đã lưu, set mặc định
export function setLocale(locale: 'vi' | 'en'): void;  // đổi + lưu vào app_meta
export function t(key: string, params?: Record<string, unknown>): string;

// src/shared/i18n/use-t.ts
export function useT(): (key: string, params?: Record<string, unknown>) => string;
```

Component **luôn** gọi `useT()`/`t()`; **cấm** chuỗi UI hard-code (NFR-9, FR-I1).

## Quy tắc & kiểm thử

- **Không sót khóa**: test `keys.test.ts` so tập khóa `en` vs `vi` — thiếu/thừa là **fail**.
- **Fallback an toàn**: thiếu bản dịch → rơi về `en`, không hiển thị key trần cho người dùng.
- **Review checklist**: khi thêm màn hình mới, thêm khóa cho **cả** `vi` và `en` trong cùng PR.
- **Đổi ngôn ngữ tức thì**: `setLocale` cập nhật store → UI render lại; không cần khởi động lại app.
- (Ghi nhận) `getLocales()` là đồng bộ; trên Android nên đọc lại khi app trở lại foreground vì người
  dùng có thể đổi ngôn ngữ hệ thống giữa chừng.
